# DM-инструменты — чеклист «что обязательно нужно за столом»

Карта потребностей мастера для проведения сессии. НЕ план реализации — список
«что вообще должно быть» по приоритетам, чтобы виден полный объём. Реализация —
поэтапно (см. dm-layer-map.md для DM-слоя действий).

Решения по доставке контента (приняты):
- ЗВУК: DM транслирует реальный аудиопоток (как микрофон). Библиотека треков —
  локальный плеер у DM; DM проигрывает трек у себя и подмешивает в свой исходящий
  WebRTC-поток. Сервер аудио НЕ дирижирует и НЕ хранит.
  СЛЕДСТВИЕ (ограничение, принять осознанно): игрок не регулирует музыку отдельно
  от голоса мастера (один поток), нет индивидуальной громкости и синхронизации
  позиции трека. Для «себя с друзьями» приемлемо.
  → audio в room-schema перестаёт быть центральным: состояние плеера живёт у DM
    локально, не в комнате. Максимум — флажок «играет музыка» для индикатора.
- КАРТЫ/ТРЕКИ: из готовой предзагруженной библиотеки (набор карт-картинок и
  аудиотреков, поставляется с приложением). Загрузку своих файлов можно добавить
  позже; сначала — выбор из библиотеки.

ОБЯЗАТЕЛЬНОЕ — без этого сессию не провести

КАРТЫ И СЦЕНЫ
- [ ] Карта-картинка (не только пустая сетка) — фон сцены из библиотеки.
- [ ] Несколько карт в игре + переключение активной («таверна» → «подземелье»).
- [ ] Выравнивание сетки по картинке (масштаб/смещение сетки) — у чужих карт
      клетки не совпадают с 5фт по умолчанию. Критично для боя на клетках.
- [ ] (позже) Загрузка своей карты-картинки.

ТУМАН ВОЙНЫ
- [ ] Скрывать неисследованные области; открывать по мере движения партии.
- [ ] fog уже в room-schema. Игроки видят ровно открытое, не больше.
- [ ] Кисть DM: открыть/закрыть область.

NPC И ВРАГИ (блок 2 DM-слоя)
- [ ] Ставить фишки NPC/врагов (портрет на фишке, статблок по клику).
- [ ] Быстро плодить копии («+3 гоблина» из одного шаблона).
- [ ] controlledBy на токенах: NPC двигает только DM (сервер проверяет права).

ИНИЦИАТИВА И БОЙ (пункт 4 плана)
- [ ] Трекер инициативы: очередь ходов, чей сейчас ход, кнопка «след. ход».
- [ ] Переключение режима Исследование/Бой (индикатор слева — сейчас заглушка).
- [ ] combat уже в room-schema. NPC со статблоком — участники трекера.

ПРОВЕРКИ И СПАСБРОСКИ (блок 1 DM-слоя — уже спроектирован)
- [ ] DM просит проверку (кого/что/DC) → игрок кидает → результат + сравнение с DC.
- [ ] Переброс за вдохновение; окно сворачивается, но не закрывается.

HP И СОСТОЯНИЯ
- [ ] Менять HP токенам (свои персонажи и NPC).
- [ ] Вешать состояния (отравлен, без сознания, оглушён…) — видимый маркер на токене.
- [ ] Маркер смерти/выбывания.

ВАЖНОЕ — сильно улучшает игру
═══════════════════════════════════════════════════════════════════════

ЗВУК (трансляцией через WebRTC, библиотека у DM)
- [ ] Фоновая музыка: один трек в цикле, плавная смена (эмбиент таверны/боя).
- [ ] Эмбиент-петли: несколько одновременно (дождь + толпа + костёр).
- [ ] Разовые эффекты: сыграть один раз, не зациклено (гром, рёв, дверь).
- [ ] Микшер: музыка + 2-3 эмбиента + кнопки эффектов, раздельная громкость
      слоёв У DM (игрокам приходит уже смикшанный поток).
- [ ] Плеер подмешивает выбранный трек в исходящий аудиопоток DM.

НАГРАДЫ И ПРЕДМЕТЫ
- [ ] Дать игроку предмет/золото в инвентарь (смыкается с инвентарём персонажа).

ЗАМЕТКИ DM
- [ ] Приватные заметки, видны только мастеру (секреты, подготовка к сцене).

СКРЫТНЫЕ БРОСКИ (блок 4, приватный канал)
- [ ] Бросок/проверка, результат виден только DM (или DM + один игрок).

ЖЕЛАТЕЛЬНОЕ — комфорт, не блокирует игру
═══════════════════════════════════════════════════════════════════════
- [ ] Раздаточные изображения («вот что вы видите» — картинка всем).
- [ ] Пинг/указка на карте (ткнуть точку, всем видно).
- [ ] Измерение расстояния между клетками.
- [ ] Шаблоны зон заклинаний (конус/сфера/линия) для прицеливания.
- [ ] Быстрый бестиарий (готовые статблоки монстров для NPC).
- [ ] Закладки сцен (быстрый прыжок к подготовленным локациям).

═══════════════════════════════════════════════════════════════════════
СВЯЗИ С ТЕКУЩИМ ПЛАНОМ (что на что опирается)
═══════════════════════════════════════════════════════════════════════
- Карты/сцены/туман → room-schema (есть fog/audio/scene), управление сценой =
  блок 3 DM-слоя (этап 4).
- NPC/HP/состояния → токены в клетках (пункт 3) + рендер карточки; блок 2 (этап 3).
- Инициатива/бой → пункт 4 плана; режим боя оживает в блоке 3.
- Проверки → блок 1 (этапы 1-2), опирается на карточки из БД + вдохновение.
- Звук → отдельный слой поверх WebRTC-голоса; библиотека треков; не зависит от
  серверного состояния (трансляция).
- Награды → инвентарь персонажа (схема), смыкается с раздачей предметов.

ПРИОРИТЕТ ДЛЯ ИГРЫ: карты + туман + NPC + инициатива + проверки = минимальный
набор, чтобы реально провести бой. Звук и награды — следом. Остальное — комфорт.
